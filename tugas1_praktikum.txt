1. Jelaskan apa yang dimaksudh dengan dokumen root?
Dokumen root adalah file atau direktori utama dalam sebuah sistem yang menjadi titik awal akses ke konten atau aplikasi lainnya.
Dalam konteks web server, dokumen root biasanya adalah file HTML atau indeks yang pertama kali dibuka saat seseorang mengakses website

2. Pada apache web server, apakah dokumen root harus terletak dalam direktori c:\xampp\htdocs? jelaskan
Tidak harus. Dokumen root Apache dapat diatur untuk terletak di direktori mana pun, tidak harus di C:\xampp\htdocs.
Namun, secara default, XAMPP mengatur dokumen root di C:\xampp\htdocs. Anda dapat mengubah lokasi dokumen root dengan mengedit file konfigurasi ApacheÂ (httpd.conf).

3. jelaslkan apa yang dimaksud dengan perintah git : init, clone, remote, add, commit, push dan chekout
* git init
Digunakan untuk menginisialisasi repository Git baru dalam folder proyek.
Setelah perintah ini dijalankan, folder .git akan dibuat dan Git mulai melacak perubahan dalam proyek tersebut.

* git clone
Digunakan untuk menyalin repository dari GitHub atau GitLab ke komputer lokal.
Perintah ini akan mengunduh seluruh repository ke dalam direktori lokal.

* git remote
Digunakan untuk menghubungkan repository lokal dengan repository jarak jauh (remote repository).

* git add
Digunakan untuk menambahkan perubahan file ke dalam staging area sebelum dilakukan commit.

* git commit
Digunakan untuk menyimpan snapshot dari perubahan yang telah ditambahkan ke staging area.
Setelah commit, perubahan tersimpan secara lokal tetapi belum dikirim ke repository remote.

* git push
Digunakan untuk mengirim commit dari repository lokal ke repository remote (GitHub/GitLab)
Perintah ini akan mengunggah commit ke branch main di GitHub/GitLab.

* git checkout
Digunakan untuk berpindah ke branch lain atau commit tertentu.

4. jelaskan perbedaan antara github dan GitLab

GitHub: GitHub menyediakan platform berbasis cloud untuk hosting repositori Git. Anda juga dapat mengelola repositori pribadi dengan fitur berbayar.
GitLab: Selain menyediakan layanan berbasis cloud, GitLab juga menawarkan versi self-hosted yang memungkinkan Anda untuk menginstal GitLab di server pribadi Anda.

5. Apa bedanya variable user dengan variable system?
Variabel User: Berlaku hanya untuk pengguna tertentu dan tidak mempengaruhi pengguna lain di komputer yang sama.
Variabel System: Berlaku untuk semua pengguna dan proses di komputer tersebut.

6. Apa persamaan variable system dengan variable constant
Variabel system dan variabel konstanta (constant) memiliki persamaan berikut:
Menyimpan Nilai: Keduanya digunakan untuk menyimpan nilai yang dapat digunakan dalam program atau sistem.
Dapat Diakses Secara Global: Keduanya sering diakses secara global oleh program atau proses yang membutuhkannya.
Stabilitas: Nilai yang disimpan dalam kedua variabel ini biasanya diharapkan untuk tidak berubah selama eksekusi program atau operasi sistem. Variabel konstanta secara definisi tidak berubah, sedangkan variabel system biasanya jarang diubah setelah diatur.
